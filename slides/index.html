libGDX

  • Crossplattform game development framework
    • Desktop: Java SE (Linux, OS X, Windows, Steam)
    • Android-based systems like Android Phones & Tablets, Android TV, OUYA and Amazon Fire TV
    • iOS
    • Web (HTML5)
  • Write once, run everywhere compile for each platform

The Difference between libGDX and Unreal Engine/Havoc/Unity3D etc

libGDXUnreal/Havoc/Unity3D
2D only2D & 3D, with emphasis on 3D
OpenGLOpenGL and DirectX
Desktop, Android, iOSDesktop, Consoles, (Android, iOS)
Java-basedBased on proprietary scripting languages or Python/LUA
API, basic toolingEditor-based
Easy to use for Java developersSteep learning curve, proprietary systems
Apache LicenseProprietary

libGDX features

  • Crossplattform features:
    • Graphics (Direct OpenGL ES 2.0)
      • Meshes, textures, shaders, ...
      • Particle system
      • 2D scene-graph and UI library
      • (Basic 3D support)
    • Physics (Box2D, Box2D-lightning)
    • Aspect entity framework (Ashley)
    • AI framework (Steering Behaviors, Formation Motion, Pathfinding, Behavior Trees and Finite State Machines)
    • Input: Keyboard, Mouse, Controllers
    • Audio output
    • File IO, Fonts and Utilities
    • Integrations for Game Services, IAP, Ads, ...
